# Cambridge exam formats — referencia oficial

Este documento resume la **estructura oficial** de cada examen Cambridge según los
Handbooks for Teachers publicados por Cambridge University Press & Assessment.

⚠️ **Este documento describe únicamente formato (número de partes, tipos de tarea,
tiempos, puntuación). NO contiene contenido de exámenes reales de Cambridge.**

El contenido de los mocks de Acertlio (textos, preguntas, opciones, respuestas)
es 100% original. Los handbooks se usan solo como referencia estructural para
que los mocks de Acertlio sigan el mismo formato que el examen real.

## Question types soportados en Acertlio

| Enum value                    | Descripción                                        |
|-------------------------------|----------------------------------------------------|
| `multiple_choice`             | Reading/Listening con 3 o 4 opciones               |
| `multiple_choice_cloze`       | Cloze con multiple choice por gap                  |
| `open_cloze`                  | Cloze abierto (rellenar palabra)                   |
| `word_formation`              | Formar palabra derivada del stem                   |
| `key_word_transformation`     | Reescribir frase usando palabra clave (B2+)        |
| `gapped_text`                 | Colocar párrafos/frases en huecos                  |
| `multiple_matching`           | Emparejar preguntas con textos/párrafos            |
| `cross_text_multiple_matching`| Comparar opiniones entre textos (solo C1)          |
| `writing_task`                | Essay, article, email, report, review              |

## Puntuación total y Cambridge Scale

Cada examen se convierte a la Cambridge English Scale (140–190 para B2, 160–210 para C1, etc.).
La conversión oficial la mantiene Cambridge en secreto y se recalibra cada sesión, así que
Acertlio usa un mapping aproximado basado en el porcentaje de aciertos:

- 60% de aciertos → borderline pass (grade C)
- 70% → grade B
- 80% → grade A

Este mapping se documenta en `lib/scoring/cambridge-scale.ts` cuando lo implementemos.
